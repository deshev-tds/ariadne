# Offsec: Consult Before Registering a Guided Plan

Applies When:
- starting a guided Offsec run
- the target picture is still unclear
- a true replan is needed after target drift

Prefer:
- call offsec_consult before registering or re-registering the plan
- carry the returned corpus framing into the plan note and book shortlist
- let methodology narrowing happen before bounded execution

Avoid:
- registering a guided plan before the target picture is narrowed
- starting run_command loops without a fresh consult after material target drift

Signal:
- new guided run without stable target framing
- target picture materially changed since the last plan

Status:
- promoted
